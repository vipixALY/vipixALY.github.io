Capitolo 4 - Algoritmi e Programmazione :

4.1 Fondamenti degli Algoritmi

Un algoritmo è una procedura step-by-step per risolvere problemi con queste caratteristiche:

  • Input: Dati in ingresso (0 o più valori)
  • Elaborazione: Passi di trasformazione ben definiti
  • Output: Risultato finale (1 o più valori)
  • Deterministico: Stessi input producono sempre stessi output
  • Finito: Termina dopo un numero finito di passi

Esempio: Algoritmo Media Numeri

Inizio
↓
Leggi N numeri
↓
Calcola somma
↓
Dividi per N
↓
Stampa media
↓
Fine

Pseudocodice corrispondente

Algorithm CalcolaMedia
Begin
    Input N  // Numero di valori
    somma ← 0
    
    FOR i FROM 1 TO N DO
        Input numero
        somma ← somma + numero
    ENDFOR
    
    media ← somma / N
    Output media
End

4.3 Pseudocodice

Linguaggio informale per descrivere algoritmi con caratteristiche:

  • Struttura simile a linguaggi di programmazione reali
  • Indipendente da linguaggi specifici
  • Facile da comprendere e tradurre in codice
  • Combina linguaggio naturale e strutture di controllo

Esempio: Conversione Celsius-Fahrenheit

Algorithm ConversioneTemperatura
Begin
    Input gradi_celsius
    gradi_fahrenheit ← (gradi_celsius × 9/5) + 32
    Output gradi_fahrenheit
End

Esempio: Calcolo Fattoriale

Algorithm CalcoloFattoriale
Begin
    Input numero
    fattoriale ← 1
    
    // Verifica per numeri negativi
    IF numero < 0 THEN
        Output "Errore: numero negativo"
        Exit
    ENDIF
    
    // Calcolo del fattoriale
    FOR i FROM 1 TO numero DO
        fattoriale ← fattoriale × i
    ENDFOR
    
    Output fattoriale
End

4.4 Strutture di Controllo

Struttura Flow Chart Pseudocodice Descrizione
Sequenza Rettangoli collegati linearmente Istruzioni in ordine Esecuzione lineare passo dopo passo
Selezione Rombo con due uscite (SI/NO) IF...THEN...ELSE Scelta tra due percorsi alternativi
Iterazione Freccia di ritorno al passo precedente WHILE...DO o FOR...TO Ripetizione di passi fino a condizione

Pseudocodice corrispondente

Algorithm SistemaLogin
Begin
    Input username
    
    IF username è registrato THEN
        Input password
        IF password è corretta THEN
            Output "Accesso consentito"
        ELSE
            Output "Accesso negato"
        ENDIF
    ELSE
        Output "Username non valido"
    ENDIF
End

4.5 Esercizi Pratici

Esercizio 1: Flow Chart Distributore Bevande

Progetta il flow chart per un distributore automatico che:

  1. Accetta monete da 0.50€, 1€ (massimo 3€)
  2. Permette di selezionare una bevanda (1-3) con prezzi diversi
  3. Verifica il credito sufficiente
  4. Se credito insufficiente, chiede di inserire altre monete
  5. Eroga la bevanda e calcola il resto
  6. Restituisce le monete se l'utente annulla l'operazione
Suggerimento: Inizia identificando i punti di decisione principali: - Credito sufficiente? - Bevanda valida? - Utente vuole continuare?

Esercizio 2: Converti in Python

Converti il seguente pseudocodice in codice Python:

Algorithm PariDispari
Begin
    Input numero
    IF numero % 2 = 0 THEN
        Output "Pari"
    ELSE
        Output "Dispari"
    ENDIF
End
# Scrivi qui la tua soluzione in Python

# Soluzione di esempio:
numero = int(input("Inserisci un numero: "))
if numero % 2 == 0:
    print("Il numero è pari")
else:
    print("Il numero è dispari")

Esercizio avanzato:

Modifica il programma per:

  1. Verificare che l'input sia un numero intero
  2. Gestire eventuali errori di input
  3. Aggiungere un ciclo per permettere all'utente di inserire più numeri

Esercizio 3: Calcolatrice Scientifica

Progetta l'algoritmo (pseudocodice e flow chart) per una calcolatrice che:

  1. Chiede all'utente di selezionare un'operazione (+, -, *, /, radice, potenza)
  2. Chiede i numeri necessari (1 o 2 a seconda dell'operazione)
  3. Esegue il calcolo
  4. Mostra il risultato
  5. Chiede all'utente se vuole eseguire un altro calcolo
Punti chiave: - Usa una struttura di selezione annidata per gestire le diverse operazioni - Gestisci i casi speciali (divisione per zero, radice di numero negativo) - Usa un ciclo per permettere più operazioni
